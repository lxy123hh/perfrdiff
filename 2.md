# 方案二当前情况、完整实验流程与后续步骤

## 1. 当前实验结论

截至目前，方案二已经验证了两件事：

1. **方案二的第一阶段是可行的**
   - 已经把 `Wav2Vec2` 音频表征接入 PerFRDiff。
   - 第一阶段 `train_diffusion.py` 可以启动、训练、保存 checkpoint。
   - 这说明“用更强的音频表征替代原始 MFCC”这个方向在工程上是成立的。

2. **方案二直接配原始第二阶段 full-rank PWSG，在当前硬件上不可行**
   - 第二阶段 `train_rewrite_weight.py` 已经成功加载第一阶段的：
     - `DiffusionPriorNetwork`
     - `TransformerDenoiser`
   - 但在 `optimizer_hypernet.step()` 处出现显存溢出：
     - `torch.cuda.OutOfMemoryError`
     - GPU 显存约 `24GB`
   - 这说明**问题不在 wav2vec 本身**，而在**第二阶段仍沿用原论文的 full-rank 权重编辑器**，显存开销过大。

因此，当前最合理的结论是：

- **方案二本身没有作废**
- 作废的是：
  - “方案二 + 原始 full-rank 第二阶段” 这一条训练路线
- 现在最推荐的路线是：
  - **方案一 + 方案二联合**
  - 即：
    - 第一阶段：`wav2vec2`
    - 第二阶段：`low-rank PWSG`

## 3. 已经遇到过的问题与原因

### 3.1 transformers 版本过高

问题：

- 服务器环境中：
  - `torch==2.0.0`
  - `transformers==5.2.0`
- 二者不兼容
- 导致 `Wav2Vec2Model` 无法实例化

解决：

```bash
pip uninstall -y transformers tokenizers
pip install --no-cache-dir transformers==4.30.2 tokenizers==0.13.3
```

### 3.2 服务器无法联网下载 HuggingFace 模型

问题：

- 服务器无法访问 `https://huggingface.co`
- 直接使用：
  - `facebook/wav2vec2-base-960h`
  会反复重试下载，最终失败

解决：

在可联网机器上下载：

```bash
python -c "from huggingface_hub import snapshot_download; snapshot_download(repo_id='facebook/wav2vec2-base-960h', local_dir='wav2vec2-base-960h')"
```

然后把整个目录上传到服务器，例如：

```text
./pretrained/wav2vec2-base-960h
```

配置改成：

```yaml
pretrained_model_name_or_path: ./pretrained/wav2vec2-base-960h
local_files_only: True
```

### 3.3 第一阶段训练时间很长

原因：

- 原始 waveform
- wav2vec 编码
- clip_length 很长
- 一个 epoch 就很慢

实际观察：

- 后期 loss 已经明显平台期
- 继续训练到很后面，收益不大

结论：

- 第一阶段不必死跑到 `500`
- 在已经进入平台期时，可以选择最近保存点直接进入第二阶段

### 3.4 第二阶段 full-rank OOM

问题：

- 第二阶段运行到：
  - `optimizer_hypernet.step()`
  时出现 OOM

原因：

- 当前 `rewrite_weight_wav2vec.yaml` 仍是：
  - full-rank `ModifierNetwork`
- 即使 `batch_size=1`
- 在 24GB 显存上仍然过重

结论：

- **第二阶段必须换成 low-rank**
- 也就是：
  - `freeze_main_net: True`
  - `use_low_rank: True`
  - `rank: 8` 或 `4`

## 6.5 第二阶段训练前必须修改

当前 `configs/rewrite_weight_wav2vec.yaml` 必须改成**低秩版**。

重点改 `main_model.args`：

```yaml
main_model:
    type:
        MainNetUnified
    args:
        input_dim: 512
        latent_dim: 1024
        embed_dim: 512
        regularization: False
        regular_w: 0.0
        num_shared_layers: 2
        modified_layers:
            [diffusion_decoder.model.decoder.layers.4.multihead_attn,
             diffusion_decoder.model.decoder.layers.6.multihead_attn,
             diffusion_decoder.model.to_emotion_feat]
        predict: shift
        modify: all
        freeze_main_net: True
        use_low_rank: True
        rank: 8
        lora_alpha: 8.0
        resume:
```

如果仍然显存吃紧，再改成：

```yaml
        rank: 4
        lora_alpha: 4.0
```
